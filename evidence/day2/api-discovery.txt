9. AUTHENTICATION OBSERVATIONS
================================================================================

Auth Mechanism: Session/Cookie-based (Django session auth)
  - CSRF token required for POST/PATCH/PUT/DELETE
  - CSRF token obtained from GET /auth/get-csrf-token/
  - Session cookie set after POST /auth/sign-in/
  - HTTP 401 returned for unauthenticated requests to protected endpoints

API Key Support: CONFIRMED
  - Environment variable: API_KEY_RATE_LIMIT (60/minute)
  - Endpoint: /api/v1/users/api-tokens/ (from Django URL conf)
  - JS reference: /api/users/api-tokens/
  - API v1 public endpoints available under /api/v1/

Auth Boundary:
  - /api/instances/ - PUBLIC (no auth)
  - /auth/get-csrf-token/ - PUBLIC
  - /api/users/me/ - 401 when unauthenticated
  - /api/workspaces/ - 401 when unauthenticated
  - /spaces/ - PUBLIC (separate app)
  - /god-mode/ - PUBLIC (separate app, has own auth)

Day 3 Recommendation:
  Session-based authentication (login via UI, capture cookies)
  OR
  User manually creates API Key via Plane UI at Settings -> API Tokens
  Then use X-API-Key header for API requests

NOTE: API Key creation endpoint exists but requires authentication first.
      The user must create an API Key through the UI or we must first
      authenticate with session to call POST /api/v1/users/api-tokens/
